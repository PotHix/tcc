Desenvolvimento de jogos utilizando as tecnologias dos navegadores compatíveis com HTML5

Willian Araujo Molinari

Motivação

  • Mostrar o estado atual da tecnologia e das ferramentas.
  • Mostrar que o HTML5 consegue suprir as necessidades básicas de desenvolvimento de um jogo casual.
  • Contribuir com informações para o desenvolvimento e crescimento do HTML5 como plataforma para jogos.

Metodologia

  • Mostrar a viabilidade da plataforma para a criação de jogos casuais.
  • Facilitar a proliferação de jogos que podem ser jogados no computador e no celular utilizando o mesmo código.
  • Dar uma visão do que compõe o HTML5.
  • Mostrar a evolução do HTML5.

Objetivos

  • Comparar com as outras tecnologias utilizadas no mercado.
  • Mostrar sua viabilidade para desenvolvimento de jogos casuais simples.

Funcionalidades do HTML5

  • Canvas
  • Transformações CSS3
  • SVG
  • Websockets
  • Offline cache
  • Local Storage
  • Audio
  • WebGL

Canvas

  • Uma API que permite ao desenvolvedor fazer desenhos diretamente no navegador.
  • Criação de formas geométricas.
  • Importação de imagens.
  • Suporte nos navegadores modernos.

DOM e transformações CSS3

  • Possibilidade de desenvolvimento de jogos sem a utilização de canvas.
  • Compatível com grande parte dos navegadores por seguir o padrão atual de criação de páginas de internet.
  • Evita o redesenho de objetos a cada frame, bastando reposicionar os atuais.
  • Utilização de efeitos e transições CSS3 aceleradas por hardware.

SVG

  • Utilização de graficos vetoriais diretamente no navegador.
  • Animações diretamente no navegador.

Websockets

  • Sockets no navegador.
  • Possibilidade de manter uma conexão direta com o servidor.
  • Troca de informações mais rapidamente.
  • Utilização para jogos multiplayer.

Offline cache

  • Jogos disponíveis sem a necessidade de acesso constante a internet.
  • Dá ao jogador a possibilidade de jogar em qualquer lugar.

Local storage

  • Local para guardar dados no navegador do usuário.
  • Substituição para o uso indevido de cookies.
  • Um ótimo local para guardar informações sobre o jogo.

Audio

  • Implementação nativa para tocar audio no navegador.
  • Grande suporte nos navegadores.
  • WebAudioApi e o audio em tempo real.
  • Subsídio para produzir toda a sonoplastia necessária para um jogo casual.

WebGL

  • Aceleração 3D via navegador.
  • Acesso a GPU utilizando o navegador.
  • Baseado em OpenGL ES.
  • Suporte grande parte dos navegadores incluindo suporte.
